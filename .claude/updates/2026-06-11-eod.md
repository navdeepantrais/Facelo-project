# EOD Report — 11 June 2026

## Bug Fixes

### Server Actions Async Constraint

- **Problem:** `lib/actions/products.ts` exported synchronous helpers under `'use server'`, causing a Next.js runtime crash ("Server Actions must be async functions").
- **Fix:** Extracted all sync helpers (`productCategorySelect`, `mapProductRow`, `parseProductFilters`, `ProductRowWithCategory`) into a new file `lib/queries/products.ts` (no directive). Updated all import sites: `lib/actions/marketplace.ts`, `lib/actions/orders.ts`, `app/(marketplace)/categories/[slug]/page.tsx`, `app/(marketplace)/products/page.tsx`.
- **Rule established:** `'use server'` files export only async functions. Sync utilities go in `lib/queries/` or `lib/utils.ts`.

### Image Hostname

## Security — RBAC Hardening

**Problem:** User could place an order without being logged in. Checkout had no auth guard.

**Changes:**

- Added `/checkout` to `PROTECTED_PREFIXES` in `proxy.ts` (edge-level)
- Created `app/(marketplace)/checkout/layout.tsx` with `requireAuth('/checkout')`
- `app/(marketplace)/checkout/page.tsx` — switched to `requireAuth`
- `app/(marketplace)/checkout/CheckoutForm.tsx` — removed `userId` prop entirely
- `lib/actions/orders.ts` — `createOrder` now gets `userId` from session internally; never accepts it from caller
- `getUserOrders` — verifies `user.id === userId` before returning data
- `app/(marketplace)/checkout/confirmation/[id]/page.tsx` — always requires auth; `getOrder(id, user.id)` enforces ownership
- `app/auth/login/page.tsx` + `app/auth/register/page.tsx` — redirect authenticated users to `/`

**Defense layers:** `proxy.ts` (edge) → layout `requireAuth` → page `requireAuth` → server action session check.
